Virtual Folders (My Notebooks)

Overview

The "My Notebooks" tree-view feature allows you to organize markdown files into a customizable hierarchical structure for easy access and navigation.

Key Features

  • Virtual Folders: Create custom folder hierarchies
  • Custom Display Names: Rename files and folders
  • Hierarchical Structure: Create nested folders
  • Quick Access: Access documents with one click

Managing Folder Groups and Virtual Folders

Creating Folder Groups
  1. Click on the New Folder Group button in the top action bar of the My Notebooks view
  2. Enter a name for the folder group when prompted
  3. If the .vscode directory or .vscode/codebook-md.json file doesn't exist, you'll be asked to create it
  4. The new folder group will appear in your My Notebooks view
Adding Virtual Folders
  1. Click on the New Folder icon for a specific folder group
  2. Enter the folder name
  3. Add files to the folder

Configuration File

Virtual folder configurations are stored in .vscode/codebook-md.json within your workspace. This file contains all your folder groups, their hierarchical structure, and file references.

  • Version Control: The configuration file can be committed to version control for team sharing
  • Manual Editing: Advanced users can manually edit the configuration file
  • Workspace-Specific: Each workspace has its own configuration file

Note: Changes made through the UI are automatically saved to this file.

Creating New Notebooks

Overview

CodebookMD makes it easy to create new interactive notebooks that combine documentation and executable code. There are multiple ways to create new notebooks based on your workflow.

Key Features

  • Multiple Creation Methods: Create notebooks from the command palette, context menu, or My Notebooks view
  • Creation from Selection: Convert selected code or text into a new notebook
  • Template Structure: New notebooks include helpful starter content
  • Seamless Integration: Notebooks are automatically opened in the CodebookMD editor

Creation Methods

There are several ways to create a new CodebookMD notebook:

From Command Palette
  1. Press Ctrl+Shift+P (Windows/Linux) or Cmd+Shift+P (macOS) to open the Command Palette
  2. Type "CodebookMD" to see available commands
  3. Select CodebookMD: Create New Notebook to create a blank notebook
  4. Select CodebookMD: Create Notebook from Selection to convert selected text into a notebook
From Context Menu
  1. Select text or code in an editor
  2. Right-click to open the context menu
  3. Select Create CodebookMD Notebook from Selection
From My Notebooks View
  1. Click on the CodebookMD icon in the Activity Bar to open the My Notebooks view
  2. In the top action bar, click either:
    • The New Notebook button to create a blank notebook
    • The From Selection button to create a notebook from your current selection

Creating a Blank Notebook

When creating a new blank notebook:

  1. You'll be prompted to enter a name for your notebook file
  2. You can select a location to save the notebook
  3. A default template will be created with example content
  4. The notebook will automatically open in the CodebookMD editor

Creating a Notebook from Selection

When creating a notebook from selected text or code:

  1. First, select the text or code you want to include in your notebook
  2. Trigger the "Create Notebook from Selection" command via any of the methods above
  3. Enter a name for your new notebook file
  4. The selected content will be formatted appropriately in the new notebook
  5. The notebook will automatically open in the CodebookMD editor

Default Template Structure

New CodebookMD notebooks include the following template structure:

# Notebook Name

## Introduction

This is a new CodebookMD notebook. You can add content and executable code blocks below.

## Code Blocks

### Example Markdown

You can write regular markdown content with formatting like **bold**, *italic*, or `code`.

### Example Code Block

```shellscript
# This is an example code block
echo "Hello from CodebookMD!"
# Click the play button to run this code
```

## Quick Tips

- Use the gear icon (⚙️) to configure code blocks
- Hover over file links to preview their content
- Add this notebook to your My Notebooks view for quick access

Adding a Notebook to My Notebooks View

After creating a notebook, you can add it to your My Notebooks view for easy access:

  1. Go to the My Notebooks view in the Activity Bar
  2. Click the "+" button next to an existing folder group or create a new folder group
  3. Select your notebook file to add it to the folder group

Best Practices

  • Descriptive Names: Use clear, descriptive names for your notebooks to make them easy to find
  • Organize in Sections: Structure your notebooks with clear headings and sections
  • Combine Code and Documentation: Pair code blocks with explanatory text for better understanding
  • Add to My Notebooks: Add frequently used notebooks to your My Notebooks view for quick access
  • Use File Links: Reference other files using file links to create connected documentation

Tips for Effective Notebooks

  • Start with a clear introduction explaining the notebook's purpose
  • Add executable examples that demonstrate concepts
  • Include both success and error cases in examples
  • Keep code blocks focused on a single task or concept
  • Add comments to explain complex code segments
  • Use proper markdown formatting for readability

Dynamic Folders (Current Context)

Overview

The "Current Context" feature automatically creates a dynamic folder group based on your active file, showing relevant markdown files in nearby directories to enhance contextual navigation.

Key Features

  • Context-Aware: Automatically displays folders containing markdown files related to your current file
  • Dynamic Updates: Changes as you navigate between different files
  • Directory Hierarchy: Shows parent directories containing markdown files
  • Visual Distinction: Visually different from regular folder groups to indicate its dynamic nature
  • Configurable: Customize behavior through settings.json

How It Works

  1. When you open or switch to a markdown file, the dynamic folder group updates automatically
  2. The extension traverses up the directory tree from your current file
  3. Any directory containing markdown files is included in the dynamic folder group
  4. Files are listed with readable names for quick navigation

Configuration Options

You can customize the dynamic folder group through VS Code settings (settings.json):

{
  "codebook-md": {
    "dynamicFolderGroup": {
      "enabled": true,
      "name": "Relevant Docs",
      "description": "Relevant docs for the current file",
      "subFolderInclusions": [
        ".github",
        ".vscode",
        "apiplayground"
      ],
      "exclusions": [
        "node_modules",
        "out",
        "dist"
      ]
    }
  }
}
Configuration Properties
  • enabled: Enable or disable the dynamic folder group feature (default: true)
  • name: Custom name for the dynamic folder group (default: Current Context)
  • description: Description shown when hovering over the folder group (default: Auto-generated based on the current file)
  • subFolderInclusions: Array of sub-folder names to include when searching for markdown files
    • If a folder is included in the dynamic folder group and has a sub-folder matching one of these names, that sub-folder will be searched for markdown files
    • Sub-folders are only included if they contain markdown files after applying exclusions
  • exclusions: Array of patterns to exclude from the search
    • Exclude specific folders, files, or patterns from the dynamic folder group
    • Supports simple glob patterns (e.g., *.tmp)
    • Default exclusions: ["node_modules", "out", "dist"]

Benefits

The dynamic folders feature helps you:

  • Discover Related Content: See other markdown files in the same context
  • Faster Navigation: Jump between related files without searching
  • Understand Structure: Visualize the organization of documentation around your current file
  • Filter Irrelevant Content: Exclude unwanted folders and focus on meaningful documentation

Note: Dynamic folders are temporary and not saved in your configuration file, making them perfect for quick contextual exploration without cluttering your saved folder groups.

Executable Code Blocks

CodebookMD brings interactive computing to Markdown files by letting you execute code blocks directly in your documents.

Getting Started

To execute a code block in your markdown file:

  1. Code blocks are automatically detected by CodebookMD when opening a markdown file.
  2. Here is a simple example for a shellscript code block that would be executable in CodebookMD
    3. ```shellscript
      echo "Hello, CodebookMD!"
```
  3. Click the "Run" triangle button that appears in the upper-left corner of the code block.
  4. The code will execute and display its output directly below the code block
  5. Tip: To create a new code block within a markdown file (already opened in CodebookMD), click on the "+ Code" button found above or below an existing cell in the notebook.

Supported Languages

CodebookMD supports executing code in multiple languages:

  • Go: Run code as main.go or test files
  • Shell/Bash: Execute shell commands
  • JavaScript/TypeScript: Run JS/TS code
  • Python: Execute Python scripts
  • SQL: Execute database queries
  • HTTP: Execute HTTP requests directly from code blocks

CodeBlock Configuration

CodebookMD offers a user-friendly configuration interface for code blocks, allowing you to customize execution and output settings without memorizing command syntax.

Using the Configuration Modal

The recommended way to configure code blocks is through the intuitive configuration modal:

  1. Click the gear icon () in the code block toolbar
  2. This opens a configuration panel with easy-to-use form controls for all available options
  3. Changes made in the configuration panel are saved directly to the code block metadata
  4. Click "Save Configuration" to apply your changes

Features of the Configuration Modal

  • User-friendly Interface: Simple toggles, dropdowns, and text fields for all options
  • Language-specific Options: Only shows relevant options for the current language
  • Settings Integration: Each configuration option includes a settings wheel icon () that opens VS Code settings directly filtered to that specific setting
  • Command Management: Easily browse and add available commands for your code block
  • Output Configuration: Visual controls for customizing how output is displayed
  • Execution Path: Set custom execution paths for your code blocks

Configuration Sections

The configuration modal is organized into several key sections:

  • Output Configuration: Control how execution results are displayed
    • Show/hide executed code in output
    • Control whether output is shown after execution
    • Replace existing output or append new output
    • Show/hide execution timestamp and set timezone
  • Language-specific Configuration: Options specific to the current language
  • Execution Path: Set the directory where code will be executed
  • Available Commands: Browse and add commands specific to your code block
  • Code Block Commands: View and manage commands already configured for your code block

Output Configuration

Configuring Output Globally

Set default output behavior in VS Code settings (settings.json):

{
  "codebook-md.output": {
    "showExecutableCodeInOutput": true,
    "showOutputOnRun": true,
    "replaceOutputCell": true,
    "showTimestamp": true
  }
}

Configuring Output Per Language

Override global settings with language-specific configurations:

{
  "codebook-md.go.output": {
    "showExecutableCodeInOutput": false,
    "showTimestamp": false
  },
  "codebook-md.bash.output": {
    "showTimestamp": true
  }
}

Go

Execute Go code directly in markdown with two execution modes.

Basic Usage
```go
fmt.Println("Hello, CodebookMD!")
```
VS Code Settings Configuration

Configure Go execution behavior in VS Code settings:

{
  "codebook-md.go": {
    "execType": "run",              // "run" or "test"
    "execTypeRunFilename": "main.go",
    "execTypeTestFilename": "codebook_md_exec_test.go",
    "execTypeTestBuildTag": "playground",
    "goimportsCmd": "gopls imports"  // or "goimports"
  }
}
Configuring Go Code Blocks

Configure Go code blocks easily through the configuration modal:

  1. Click the gear icon in the code block toolbar
  2. Adjust settings in the Go-specific configuration section
  3. Click "Save Configuration" to apply your changes
Go-specific Options
  • Execution Type: Choose how your code is executed
    • Run Mode (default): Executes code as a main.go file
      • Option: execTypeRunFilename - Customize the filename for run mode
      • Your code is automatically wrapped in a main function if needed
    • Test Mode: Executes code as a test file
      • Option: execTypeTestFilename - Set custom test filename
      • Option: execTypeTestBuildTag - Set build tag for tests
      • Your code is automatically wrapped in a test function
  • Import Management: Control how imports are handled
    • Option: goimportsCmd - Choose between "gopls imports" (more reliable) or "goimports" (faster)
  • Output Filtering: Filter unwanted text from output
    • Option: excludeOutputPrefixes - List of prefixes to exclude from output
  • Execution Path: Set the directory where code will be executed
Best Practices for Go
  • Add test files to .gitignore to prevent test pollution
  • Use build tags to isolate notebook tests from package tests
  • Consider using goimports for faster import resolution
  • Use output filtering for cleaner execution results

Custom CellConfig JSON Files

CodebookMD now supports storing cell configurations in separate JSON files instead of embedding them in notebook metadata. This provides better version control, easier sharing, and cleaner notebook files.

Key Features

  • External Configuration Storage: Store cell configurations in dedicated JSON files
  • Configurable Location: Customize where configuration files are stored
  • Version Control Friendly: Configuration files can be easily tracked in version control
  • Per-Notebook Configuration: Each notebook has its own configuration file
  • Language-Specific Settings: Store detailed configuration for each programming language

How It Works

When you configure a code block using the configuration modal, the settings are automatically saved to a JSON file rather than being embedded in the notebook's metadata. This keeps your notebook files clean while preserving all customization options.

Configuration File Location

By default, configuration files are stored in a codebook-md/ directory relative to your notebook. You can customize this location in VS Code settings:

{
  "codebook-md.notebookConfigPath": "./codebook-md/"
}

Supported Path Types

  • Relative Paths: ./config/ or ../shared-config/ - Resolved relative to the notebook's directory
  • Absolute Paths: /usr/local/codebook-configs/ - Use absolute filesystem paths

Configuration File Structure

Configuration files are named <notebook-name>.config.json and contain cell-specific configurations:

{
  "0": {
    "config": {
      "output": {
        "showExecutableCodeInOutput": true,
        "showOutputOnRun": true,
        "replaceOutputCell": false,
        "showTimestamp": true,
        "timestampTimezone": "America/New_York"
      },
      "execType": "run",
      "pythonCmd": "python3"
    }
  },
  "2": {
    "config": {
      "output": {
        "showTimestamp": false
      },
      "execSingleLineAsCommand": true
    }
  }
}

Configuration Structure

  • Cell Index Keys: Each top-level key corresponds to a cell's index in the notebook (0-based)
  • Config Object: Contains all configuration options for that specific cell
  • Output Settings: Control how execution results are displayed
  • Language-Specific Options: Include settings specific to the cell's programming language

Using Custom CellConfig

Automatic Configuration

Configuration files are created and managed automatically:

  1. Open any code block's configuration modal by clicking the gear icon
  2. Adjust your settings using the intuitive interface
  3. Click "Save Configuration" to apply changes
  4. Settings are automatically saved to the appropriate JSON file

Manual Configuration

Advanced users can directly edit configuration files:

  1. Navigate to your configured notebookConfigPath directory
  2. Open the <notebook-name>.config.json file
  3. Edit the JSON structure to add or modify cell configurations
  4. Save the file - changes will be automatically applied

Environment Variables

CodebookMD now supports environment variables from VS Code settings in shell scripts.

Key Features

  • VS Code Settings Integration: Shell scripts executed in CodebookMD automatically have access to environment variables set in VS Code settings
  • Platform-Specific Support: Variables defined in terminal.integrated.env.osx, terminal.integrated.env.windows, or terminal.integrated.env.linux are available to shell commands
  • Seamless Experience: Scripts run with the same environment as the integrated terminal without additional configuration

How It Works

When you execute a shell code block in a markdown file, CodebookMD automatically:

  1. Detects your current operating system (macOS, Windows, or Linux)
  2. Reads environment variables from the corresponding VS Code setting (terminal.integrated.env.*)
  3. Merges these variables with the system environment variables
  4. Makes all variables available to your executed shell commands

Setting Up Environment Variables

Configure environment variables in your VS Code settings.json file:

{
  "terminal.integrated.env.osx": {
    "API_KEY": "your-api-key",
    "DATABASE_URL": "postgres://user:password@localhost:5432/mydb",
    "PATH": "${env:PATH}:/custom/path"
  }
}

These environment variables will be automatically available in your shell code blocks:

```bash
# This will have access to API_KEY, DATABASE_URL, and the extended PATH
echo $API_KEY
echo $DATABASE_URL
echo $PATH
```

Benefits

  • Consistent Environment: The same environment variables available in your terminal are now available in code blocks
  • Secure Credential Management: Store sensitive credentials in VS Code settings instead of hardcoding them in markdown files
  • Simplified Configuration: Configure once in VS Code settings, use everywhere
  • Platform Adaptability: Scripts automatically use the correct environment variables for the current operating system

Note: Environment variables are merged at execution time, with VS Code settings taking precedence over system environment variables when there are conflicts.

Codebook Prompts

Codebook Prompts allow you to create interactive code blocks that request user input before execution.

Why Use Codebook Prompts?

  • Interactive Execution: Prompt for user input when running code blocks
  • Dynamic Parameters: Easily update dates, filters, or other values without modifying code
  • Streamlined Workflow: No need to track and change hardcoded values for repeated executions
  • Enhanced Documentation: Create more interactive and user-friendly notebooks

How to Use Codebook Prompts

Add prompt commands in your code blocks using the [>].prompt syntax.

Syntax

The syntax for Codebook Prompts is:

[>].prompt.Type(placeholder text)

Where:

  • Type is the type of prompt (String or Date)
  • placeholder text is the message shown to the user

Supported Prompt Types

  • String: For general text input 
    # Example with String prompt
echo "My favorite color is [>].prompt.String(enter your favorite color)"
  • Date: For date input (with YYYY-MM-DD validation) 
    # Example with Date prompt
echo "Report for: [>].prompt.Date(enter report date)"

Using Multiple Prompts

You can include multiple prompts in a single code block:

# Example with multiple prompts
echo "Name: [>].prompt.String(enter your name)"
echo "Birthday: [>].prompt.Date(enter your birthday)"

How It Works

  1. When you execute a code block containing prompts, CodebookMD detects them and shows input dialogs
  2. As you provide input for each prompt, the values replace the prompt syntax in the code
  3. Once all prompts are filled, the code executes with your provided values
  4. The output shows the code with your values substituted for the prompts

Best Practices

  • Use descriptive placeholder text to clearly indicate what input is expected
  • For dates, remember the required format is YYYY-MM-DD
  • Consider using prompts for values that change frequently between executions
  • Combine with environment variables for a powerful execution workflow

Examples

Shell Script Example

```shellscript
# Script to create a dated log file
FILENAME="log-[>].prompt.Date(enter log date)"

SQL Example

```sql
-- Query data within a date range
SELECT * FROM orders 
WHERE order_date BETWEEN '[>].prompt.Date(enter start date)' 
AND '[>].prompt.Date(enter end date)'
LIMIT 10;
```
